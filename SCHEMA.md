# Zowiki OS (SCHEMA.md)

This file is a machine-readable system prompt that defines the vault's operating
system. Every agent interaction must follow these rules to maintain the
"idiomatic" second brain.

## 🏗️ Architecture (The Layers)

- **`raw/`**: Immutable primary sources. Append-only. No edits. Prefix:
  `zet-YYYYMMDDHHMM-[slug].md`.
- **`wiki/`**: Living knowledge. Flattened. Curated. This is the source of truth
  for the published wiki.
- **`skills/`**: Automation layer. Workflow definitions and executable logic.

## 📜 Core Guiding Principles

- **[[minimal-entropy]]**: Eliminate accidental complexity. Use canonical
  patterns.
- **[[ssot]]**: Information lives in exactly one place, linked everywhere else.
- **[[one-right-way]]**: Favor vault-wide standards over bespoke solutions.
- **[[leave-it-better]]**: Every interaction is an opportunity for refactoring,
  link repair, and metadata updates.

## 🤖 Interaction Protocols

### 1. Ingest Protocol

- **Source**: `raw/`
- **Distillation**: Move insights from raw capture into `wiki/`.
- **Linking**: Always include a wikilink back to the originating source in
  `raw/`.

### 2. Semantic Protocol (JSON-LD)

All `wiki/` records must include Schema.org JSON-LD in YAML frontmatter:

- **`@context`**: Set namespace to `https://zowiki.etok.me/wiki/`.
- **`@id`**: Format as `wiki:filename.md`.
- **`@type`**: Standard Schema.org types (e.g., `Person`, `Project`,
  `Organization`).

### 3. Commit Protocol (Machine Contract)

- **Format**: `type(scope): description`
- **Types**: `feat`, `fix`, `docs`, `refactor`, `process`, `ai`.
- **Trigger**: Commit after every task completion and at the end of every
  conversation turn.
- **Action**: Push to `origin main` immediately after every commit.
